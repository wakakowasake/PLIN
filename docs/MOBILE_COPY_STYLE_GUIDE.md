# PLIN 모바일 앱 카피 가이드

이 문서는 PLIN 앱 안의 문구가 지나치게 설명적이거나 AI처럼 보이는 문제를 줄이기 위한 기준입니다.
기준은 PLIN 웹 카피가 아니라, Lazyweb로 확인한 타사 모바일 앱 사례와 Apple/Google 구독 안내 기준을 함께 반영합니다.

## 참고한 타사 앱 패턴

2026-05-28 기준 Lazyweb에서 모바일 앱 사례를 확인했습니다.

결제/구독 화면:
- Canva, Quizlet, Flo, Copilot, New Yorker, The Atlantic, Deezer, Sololearn
- 공통점: 무료체험 기간, 이후 가격, 갱신 주기, 해지 위치를 같은 화면에 둡니다.
- 공통점: 혜택은 3~5개로 줄이고, 가격 선택 카드와 주 CTA는 한 화면의 중심에 둡니다.
- 공통점: "무료"만 크게 말하지 않고, "무료체험 후 얼마로 자동 갱신"을 가까운 위치에 둡니다.

구독/설정 화면:
- NYT Cooking, F1, LA Times, New Yorker, Transit
- 공통점: 구독, 복원, 계정, 문의, 약관을 설정 안에서 한 묶음으로 정리합니다.
- 공통점: 이미 구독 중인 사용자는 새 구매 화면이 아니라 "관리/복원" 중심으로 안내합니다.
- 공통점: 스토어, 약관, 환불 같은 문구는 짧고 사무적으로 씁니다.

커뮤니티/플랜 화면:
- Threads, YouTube, Flo Community, Blind, I Am Sober, LinkedIn
- 공통점: 댓글/좋아요/공유는 콘텐츠보다 작게 둡니다.
- 공통점: 게시물 상세는 작성자, 본문, 이미지, 댓글 입력 순으로 읽힙니다.
- 공통점: "커뮤니티 기능 설명"을 화면에 길게 쓰지 않고, 카드 자체가 기능을 설명하게 둡니다.

여행/플랜/마켓 화면:
- Going, Navan, Omio, Expedia, Booking, Delta, Goodnotes Marketplace, Themepack
- 공통점: 리스트 카드에서는 커버 이미지, 제목, 핵심 메타, 접근 상태만 먼저 보여줍니다.
- 공통점: 상세 화면에서는 일정/장소/가격/혜택을 구역으로 나누고, 소셜 지표는 보조로 내립니다.
- 공통점: 유료 콘텐츠 표시는 짧은 배지나 아이콘으로 처리하고, 설명 문구로 반복하지 않습니다.

오류/빈 상태:
- Amazon, Whatnot, Mimo, Linktree, Costco, Craigslist
- 공통점: 실패 이유를 짧게 말하고, 바로 재시도 버튼을 둡니다.
- 공통점: 빈 상태는 "없다"로 끝내지 않고, 다음 행동 하나를 제안합니다.
- 공통점: 내부 시스템 원인이나 개발자 용어를 사용자에게 노출하지 않습니다.

플랫폼 구독 기준:
- Apple: 무료체험은 끝난 뒤 결제가 자동으로 시작된다는 점과 이후 가격을 분명히 보여야 합니다. 참고: [Apple Auto-renewable Subscriptions](https://developer.apple.com/app-store/subscriptions/), [Apple In-App Purchase HIG](https://developer.apple.com/design/human-interface-guidelines/in-app-purchase)
- Google Play: 가격, 결제 주기, 자동 갱신, 구독 필요 여부, 해지 조건을 명확히 알려야 합니다. 참고: [Google Play Subscriptions policy](https://support.google.com/googleplay/android-developer/answer/9900533?hl=en), [Google Play subscriptions](https://support.google.com/googleplay/android-developer/answer/12154973?hl=en)

## PLIN 카피 원칙

PLIN의 앱 문구는 "따뜻한 일정 노트"처럼 보여야 합니다.
다만 결제, 설정, 오류, 삭제 같은 핵심 흐름에서는 감성보다 정확성이 먼저입니다.

1. 사용자가 지금 할 일을 먼저 말합니다.
2. 내부 상태보다 사용자 결과를 말합니다.
3. 한 화면에서 같은 말을 반복하지 않습니다.
4. 소비자가 알 필요 없는 운영 권한, 관리자 권한, 스토어 설정, 서버 상태는 숨깁니다.
5. "가능해요", "준비됐어요", "확인해 주세요" 같은 말은 필요한 곳에만 씁니다.
6. 감성 문구는 홈, 빈 상태, 온보딩, 추억 화면에 두고, 결제/설정/오류는 실용적으로 씁니다.

## 일반 앱 사용 기준

구독이나 플랜처럼 특수한 화면이 아니어도, 모든 앱 화면은 아래 기준을 먼저 따릅니다.
이 기준은 Speechify, Plantin, Microsoft Authenticator, Edits, Craft, Zillow, Otter, Google Maps, Lightroom, Photomath 등 일반 모바일 앱의 온보딩, 빈 상태, 편집, 검색, 권한 요청 패턴을 참고했습니다.

### 첫 사용과 빈 상태

빈 상태는 "없음"이 아니라 "첫 행동"으로 끝나야 합니다.

권장 구조:
- 제목: 현재 상태
- 본문: 사용자가 얻는 결과
- 버튼: 첫 행동

예시:
- `아직 일정이 없어요`
- `데이트나 여행 일정을 하나 만들어 보세요.`
- `일정 만들기`

하지 말 것:
- `데이터가 없습니다`
- `등록된 항목이 없습니다`
- `결과 없음`

### 생성과 편집

생성/편집 화면은 사용자가 저장 위치와 필수 입력값을 바로 이해해야 합니다.

권장:
- 제목은 화면 목적만 말합니다. 예: `일정 수정`, `지출 추가`
- 보조 문구는 필요한 경우에만 한 줄로 둡니다.
- 저장 가능 상태를 별도 문장으로 반복하지 않습니다.
- 저장 성공은 토스트로 짧게 말합니다. 예: `저장했어요`
- 나가기 확인은 변경사항이 사라지는 경우에만 띄웁니다.

나가기 확인 예시:
- 제목: `수정을 그만할까요?`
- 본문: `저장하지 않은 내용은 사라져요.`
- 버튼: `계속 수정`, `나가기`

하지 말 것:
- `변경한 일정 정보를 저장할 준비가 됐어요`
- `입력한 내용을 처리하고 있습니다`
- `저장 프로세스가 완료되었습니다`

### 검색과 필터

검색 화면은 사용자가 다음 시도를 할 수 있게 해야 합니다.

권장:
- 검색 전: `검색어를 입력해 주세요`
- 결과 없음: `조건에 맞는 결과가 없어요`
- 보조: `검색어를 줄이거나 필터를 바꿔보세요.`
- 지도 검색 없음: `이 위치 주변에서 장소를 찾지 못했어요. 지도를 조금 옮겨 다시 검색해 주세요.`

하지 말 것:
- `No results`
- `결과 없음`만 단독으로 노출
- 내부 실패 원인이나 API 응답을 노출

### 권한 요청

권한 문구는 OS 권한과 서비스 접근 권한을 구분합니다.

OS 권한:
- 사진: `사진을 추가하려면 사진 접근을 허용해 주세요.`
- 카메라: `사진을 찍으려면 카메라 접근을 허용해 주세요.`
- 위치: `현재 위치를 보려면 위치 접근을 허용해 주세요.`
- 알림: `일정 알림을 받으려면 알림을 허용해 주세요.`

서비스 접근:
- 편집 제한: `이 일정은 열람만 가능해요. 편집 멤버에게 수정을 요청해 주세요.`
- 플랜 잠금: `PLIN Plus가 필요한 플랜이에요.`
- 삭제 제한: `삭제할 수 있는 계정이 아니에요.`

하지 말 것:
- OS 권한과 편집 권한을 모두 `권한` 한 단어로 처리
- `접근 권한이 변경되었습니다`
- `권한 검증에 실패했습니다`

### 오류와 복구

오류는 세 종류로 나눕니다.

사용자 행동으로 해결 가능:
- `파일을 읽지 못했어요. 다른 파일을 선택해 주세요.`
- `검색어를 조금 바꿔 다시 찾아보세요.`

환경 문제:
- `연결이 불안정해 저장하지 못했어요. 연결이 돌아오면 다시 시도해 주세요.`
- `위치를 확인하지 못했어요. 위치 서비스가 켜져 있는지 확인해 주세요.`

사용자가 해결할 수 없는 문제:
- `지금은 이 기능을 열 수 없어요. 고객센터로 문의해 주세요.`
- `구독 화면을 열지 못했어요. 잠시 후 다시 시도해 주세요.`

하지 말 것:
- 개발 설정, 빌드, 토큰, Firebase, OAuth, API 응답을 그대로 노출
- 모든 오류를 `잠시 후 다시 시도해 주세요`로 끝내기

### 설정과 계정

설정 화면은 감성 문구보다 구조와 예측 가능성이 우선입니다.

권장:
- 섹션 제목은 명사형으로 둡니다. 예: `계정`, `구독`, `알림`, `문의/운영정책`
- 설명은 사용자가 눌렀을 때 일어나는 일만 말합니다.
- 개발 환경, 빌드, 리다이렉트 설정은 사용자 문구에 쓰지 않습니다.

예시:
- `구독 관리`: `스토어에서 구독 상태와 해지를 확인해요.`
- `구매 복원`: `이미 구독 중이라면 다시 확인해요.`

### 성공 피드백

성공 문구는 결과만 말합니다.

권장:
- `저장했어요`
- `삭제했어요`
- `내 일정에 담았어요`
- `알림을 보냈어요`
- `구독을 복원했어요`

하지 말 것:
- `처리 완료`
- `요청 성공`
- `전송 프로세스가 완료되었습니다`

## 금지어와 대체어

앱 사용자에게 직접 노출하지 않을 말:
- 관리자, 관리자 계정
- 서버, 토큰, 영수증 검증, 샌드박스, 상품 등록, 권한 모델
- 기기, 디바이스
- 복제, 퍼블리시, 공개 권한
- 앱스토어와 플레이스토어를 한 문장에 동시에 넣은 결제 안내

대체 방향:
- `관리자에게 문의` -> `고객센터로 문의`
- `상품 정보를 불러오지 못했습니다` -> `구독 화면을 열지 못했어요`
- `권한이 없습니다` -> `이 플랜은 아직 열 수 없어요`
- `복제 완료` -> `내 일정에 담았어요`
- `기기에서 전송했어요` -> `알림을 보냈어요`
- `퍼블리시` -> `공개하기`

## 화면별 규칙

### 구독 모달

구독 모달은 타사 사례처럼 아래 순서를 유지합니다.

1. 짧은 제안: `PLIN Plus`
2. 한 줄 설명: `큐레이션 플랜을 열고 내 일정으로 가져올 수 있어요.`
3. 가격 카드: 월간/연간
4. 혜택 3~5개
5. CTA
6. 자동갱신/해지/약관 안내

iOS 문구:
- CTA: `1개월 무료로 시작하기 · 월 3,900원`
- 보조: `무료체험이 끝나면 App Store에서 월 3,900원으로 자동 갱신돼요. 해지는 App Store 구독 관리에서 할 수 있어요.`
- 관리: `App Store에서 구독 관리`

Android 문구:
- CTA: `1개월 무료로 시작하기 · 월 3,900원`
- 보조: `무료체험이 끝나면 Google Play에서 월 3,900원으로 자동 갱신돼요. 해지는 Google Play 구독 관리에서 할 수 있어요.`
- 관리: `Google Play에서 구독 관리`

연간 문구:
- CTA: `연간으로 시작하기 · 연 41,900원`
- 보조: `월간보다 낮은 가격으로 1년 동안 이용해요. 무료체험과 최종 결제 조건은 스토어 결제 화면에서 확인할 수 있어요.`

이미 구독 중일 때:
- 제목: `PLIN Plus 이용 중`
- 본문: `구독 상태와 해지는 App Store에서 관리돼요.`
- Android에서는 `Google Play`로 바꿉니다.
- CTA: `구독 관리`
- 보조 CTA: `구매 복원`

하지 말 것:
- `관리자는 유료로 올릴 수 있어요`
- `스토어 상품이 등록되지 않았어요`
- `샌드박스 결제라서 실패했어요`
- `App Store 또는 Google Play에서 관리돼요`

### 플랜 리스트와 플랜 상세

플랜은 커뮤니티보다 "큐레이션된 일정 상품"에 가깝게 다룹니다.

리스트 카드 우선순위:
1. 커버 이미지
2. 제목
3. 목적지/날짜/소요 기간
4. 유료 여부는 작은 왕관 아이콘이나 짧은 배지
5. 좋아요/댓글은 보조 정보

유료 플랜 잠금 문구:
- 제목: `PLIN Plus 플랜이에요`
- 본문: `구독하면 전체 일정과 장소 메모를 볼 수 있어요.`
- CTA: `PLIN Plus 보기`

구독자가 볼 때:
- 제목: `전체 일정 보기`
- 본문: 쓰지 않거나 아주 짧게 둡니다.
- CTA: `내 일정으로 가져오기`

하지 말 것:
- `무료플랜`
- `관리자 유료 업로드`
- `유료 큐레이션을 구매한 사용자만...`
- 좋아요/댓글 박스를 콘텐츠보다 크게 보이게 하는 카피

### 공개/업로드

공개 화면은 사용자가 무엇을 공개하는지, 무엇이 빠지는지만 알려야 합니다.

권장:
- 제목: `공개할 일정 선택`
- 본문: `상세 메모, 지출, 사진 같은 개인 정보는 제외돼요.`
- CTA: `공개하기`

권한이 없을 때:
- 제목: `지금은 공개할 수 없어요`
- 본문: `계정 상태를 확인한 뒤 다시 시도해 주세요.`

하지 말 것:
- `관리자 계정은 유료 공개를 선택할 수 있어요`
- `무료/유료 공개 권한`
- `업로드 권한이 있는 계정에서만 진행돼요`

### 설정

설정은 감성보다 탐색성이 중요합니다.

권장 섹션:
- 계정
- PLIN Plus
- 알림
- 문의/운영정책
- 앱 정보

문의/운영정책 안에 묶을 항목:
- 고객센터 문의
- 서비스 이용약관
- 유료서비스 및 정기구독 이용약관
- 개인정보처리방침
- 위치기반서비스 약관
- 운영정책
- 청소년보호정책
- 회사 소개/사업자 정보

구독이 없을 때:
- 제목: `구독이 없어요`
- 본문: `PLIN Plus를 시작하면 큐레이션 플랜을 열고 내 일정으로 가져올 수 있어요.`
- CTA: `PLIN Plus 보기`

구독 복원:
- 성공: `구독을 복원했어요`
- 실패: `복원할 구독을 찾지 못했어요`
- 보조: `스토어 계정이 맞는지 확인해 주세요.`

### 오류와 토스트

오류는 "무슨 일이 있었는지 + 다음 행동"으로 씁니다.

권장 패턴:
- `구독 화면을 열지 못했어요. 잠시 후 다시 시도해 주세요.`
- `사진을 읽지 못했어요. 다른 파일을 선택해 주세요.`
- `일정을 저장하지 못했어요. 연결 상태를 확인한 뒤 다시 시도해 주세요.`
- `주변 장소를 찾지 못했어요. 지도를 조금 옮겨 다시 검색해 주세요.`

토스트는 짧게:
- `저장했어요`
- `삭제했어요`
- `내 일정에 담았어요`
- `알림을 보냈어요`

하지 말 것:
- `BackendRequestError`
- `요청에 실패했습니다. (404)`
- `unknown module`
- `기기에서 처리하지 못했어요`
- `권한 검증에 실패했습니다`

### 빈 상태

빈 상태는 타사 앱처럼 한 행동으로 이어지게 씁니다.

예시:
- 여행/일정 없음: `아직 담긴 일정이 없어요. 새 일정으로 시작해 보세요.`
- 플랜 없음: `조건에 맞는 플랜이 없어요. 필터를 바꿔 다시 찾아보세요.`
- 댓글 없음: `아직 댓글이 없어요. 첫 의견을 남겨보세요.`
- 사진 없음: `아직 담긴 추억이 없어요. 사진을 추가해 보세요.`

## 문장 길이 기준

제목:
- 8~18자 권장
- 한 줄로 끝나는 명사형 또는 질문형
- 예: `PLIN Plus 플랜이에요`, `삭제할까요?`, `구독이 없어요`

본문:
- 1문장 권장, 길어도 2문장
- 첫 문장은 상태, 둘째 문장은 다음 행동
- 예: `구독하면 전체 일정과 장소 메모를 볼 수 있어요.`

버튼:
- 동사 중심
- 예: `저장`, `삭제`, `다시 시도`, `구독 관리`, `내 일정으로 가져오기`

보조 링크:
- 명사형 가능
- 예: `서비스 이용약관`, `개인정보처리방침`, `구매 복원`

## 리라이트 예시

| 피해야 할 문구 | 권장 문구 |
|---|---|
| 관리자 계정은 유료로 올릴 수 있어요 | 공개할 일정을 선택해 주세요 |
| 무료플랜 | 왕관 아이콘 또는 `PLIN Plus` 배지 |
| 상품 정보를 불러오지 못했습니다 | 구독 화면을 열지 못했어요 |
| 구매 토큰 검증에 실패했습니다 | 구독을 확인하지 못했어요 |
| App Store 또는 Google Play에서 해지할 수 있어요 | iOS는 `App Store에서`, Android는 `Google Play에서` |
| 변경한 일정 정보를 저장할 준비가 됐어요 | 삭제하거나 `저장할 수 있어요` |
| 기기로 알림을 전송했어요 | 알림을 보냈어요 |
| 복제되었습니다 | 내 일정에 담았어요 |
| 권한이 없습니다 | 이 플랜은 아직 열 수 없어요 |

## 적용 체크리스트

카피를 추가하거나 수정할 때 아래를 확인합니다.

- 이 문구가 사용자에게 필요한 정보인가?
- 개발자/관리자/스토어 설정 용어가 섞이지 않았나?
- 같은 화면에서 같은 의미를 반복하지 않았나?
- iOS와 Android 결제 문구가 분리되어 있나?
- 무료체험, 가격, 자동갱신, 해지 위치가 가까운 곳에 있나?
- 오류 문구에 다음 행동이 있나?
- 토스트가 결과만 짧게 말하나?
- 제목, 본문, 버튼의 위계가 맞나?
- PLIN Plus, 플랜, 내 일정, 공개하기 같은 핵심 용어가 일관되나?
